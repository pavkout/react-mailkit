[Mjml](https://mjml.io/) is the best responsive mail framework, I ‚ù§ it. It's a markup language created by [Mailjet](https://www.mailjet.com/).

I created this project to have a React library that use the mjml API to convert a mjml template into valid html ready to use.

## Options

Following are the options you can pass when rendering to HTML, these are the default values:

```js
{
  fonts: {
    'Open Sans':
      'https://fonts.googleapis.com/css?family=Open+Sans:300,400,500,700',
    'Droid Sans':
      'https://fonts.googleapis.com/css?family=Droid+Sans:300,400,500,700',
    Lato: 'https://fonts.googleapis.com/css?family=Lato:300,400,500,700',
    Roboto: 'https://fonts.googleapis.com/css?family=Roboto:300,400,500,700',
    Ubuntu: 'https://fonts.googleapis.com/css?family=Ubuntu:300,400,500,700',
  },
  keepComments: true,
  beautify: false,
  minify: false,
  minifyOptions: {},
  validationLevel: 'soft',
  filePath: '.',
  mjmlConfigPath: '.',
  useMjmlConfigOptions: false,
  juicePreserveTags: null,
  juiceOptions: {},
  skeleton: '',
  actualPath: '.',
  ignoreIncludes: false,
  preprocessors: [],
}
```

If you want to override these, you can pass an object to `useEmail` as a second argument. See the [MJML docs](https://documentation.mjml.io/#inside-node-js) for the full list of options you can set.
